映像制作の４０年と、これからの時代へ

映像制作は1980年代から大きな変革を遂げました。フィルムから磁気ベースのVTRやカメラへ、そしてフラッシュメモリーをストレージにしたカメラ・サーバー・編集システムが当たり前になり、ソリューションによってはこの30年でその価格は100分の1にまで下がりました。 クラウドへ移行するソリューションが主流となり、映像フォーマットは高圧縮しても画質が維持されるようになり、クラウドを越えたグローバルネットワークも実現しました。そして今、生成AIをはじめとしたAIの機能が映像制作ソリューションの新たな時代を切り拓こうとしています。 この30年間、映像技術のイノベーションに伴奏してきた当社のコンサルタントが、新たな映像ソリューションによるOMO・DCXの実現に貢献します。

CANTO Dev Log

一行のコードも書かずに——
ツールが生まれた日 Not a single line of code —
the day a tool was born

プログラミング経験ゼロ。それでも「作りたい」という衝動がClaudeを動かした。CANTOの誕生から進化まで、開発の舞台裏を記録したdevログ。 Zero coding experience. Yet the urge to build set everything in motion. A behind-the-scenes dev log of CANTO's birth and evolution.

V1 GENESIS — Full Player Day 1 V2–V8 UX & FEATURES V9… MORE TO COME
PHASE — 01

GENESIS — CRATEと名付けられたプロトタイプ GENESIS — The Prototype Named CRATE

↑ CRATE V1（2026年3月27日）— 初日から完全なMP3プレイヤーとして動作。再生・キュー・選択モード・お気に入りが揃っていた ↑ CRATE V1 (March 27, 2026) — Fully working MP3 player from day one. Playback, queue, selection mode and favorites were all in place
TARGET FOR PLANNING / 企画の狙い TARGET FOR PLANNING

V1 初日から完全動作——まず「使えるもの」を作り、そこから磨く V1 worked fully on day one — build something usable first, then refine

最初のプロンプトからClaudeが生成したプロトタイプ「CRATE（クレート）」は、初日から完全なMP3プレイヤーとして機能した。再生・停止・ライブラリ管理・お気に入り・選択モードにキューまで——一行のコードも書かずにこれが動いた衝撃は忘れられない。 The first prototype named "CRATE" worked as a complete MP3 player from day one — playback, library management, favorites, selection mode, and queue were all there. The shock of having all this without writing a single line of code was unforgettable.

この段階ではファイルを一曲ずつ選ぶ方式でしたが、それ以外は現在のCANTOと変わらない使いやすさを持っていた。以降のバージョンアップは「使いやすさの向上」から始まり、やがて大きな機能追加へと発展していく。 Files were selected one at a time at this stage, but everything else was already highly usable — essentially the same experience as today's CANTO. Subsequent updates started with usability improvements, then evolved into major feature additions.

Prompt / 最初のプロンプト Prompt / First Input 
USER →MP3プレーヤーを作りたいです。車やMP3プレーヤーでも利用できるようにディレクトリー構造としてルートの音楽をすべてブラウズできるようにして　そこから仮想的にフォルダーを階層化するGUIにしますが、保存場所は同じ　フォルダーごとの合計データサイズを計算して表示　再生中は曲名を表示　あとは重複した名前はデータサイズが一致したら更新が新しいものを頭に表示　お気に入りのメタデータをファイルに書き込めるようにI want to build an MP3 player. Make it usable in a car or on a standalone MP3 player — browse all music as a virtual folder hierarchy from the root. Show total data size per folder. Display the track name during playback. For duplicate filenames, show the most recently modified one first if sizes match. Allow writing favorite metadata to files.
Response / Claudeのレスポンス抜粋 Response / Claude's Key Output
「まず、フロントエンドデザインのスキルを確認してから作成します。では、高品質なMP3プレーヤーを作成します！」 "First, let me check the frontend design skill. Now let's build a high-quality MP3 player!"

ファイルを作成しました、ファイルを読み取りました——そして：
「CRATE — 完成しました！単一HTMLファイルとして保存されているので、車載プレーヤーやスタンドアロン環境でもそのまま動きます。」 File created. File read. And then —
"CRATE — Done! Saved as a single HTML file, so it works as-is in car players and standalone environments."

PHASE — 02

IDENTITY & EVOLUTION — 「CANTO」という名前の誕生と、開発一日目のフォルダー実装 IDENTITY & EVOLUTION — The Name "CANTO" and Folder Support on Day One

↑ CANTO V8（2026年3月28日）— フォルダー一括読み込み。1,326曲を一度のアクションでライブラリに追加 ↑ CANTO V8 (March 28, 2026) — Batch folder loading. 1,326 tracks added to the library in a single action
TARGET FOR PLANNING / 企画の狙い TARGET FOR PLANNING

名前が決まると、方向性が決まる——「CANTO」とGeminiとの壁打ち A name defines a direction — "CANTO" emerged from Gemini brainstorming sessions

V1の改善に向け、私はGeminiと集中的な「壁打ち」を行いました。単なるプレーヤーではなく、音楽が歌い出すような、プロフェッショナルかつ親しみやすいアイデンティティは何か？その議論の中から生まれたのが、ラテン語で「歌」「詠唱」を意味する「CANTO（カント）」という名称です。 To evolve V1, I conducted intense brainstorming sessions with Gemini. We were looking for an identity that felt like music coming to life — professional yet accessible. From these discussions emerged "CANTO", Latin for "song" or "chant."

機能面での最大のマイルストーンは、フォルダー一括読み込み機能の実装です。注目すべきは、これが開発開始からわずか一日目（2026年3月28日、V8）に実現したという事実。「一曲ずつ選ぶ」という非現実的な方式から、「フォルダーを指定するだけで1,326曲が即座にライブラリに登録される」というワークフローへ。この一歩がCANTOを実用ツールに変えた転換点でした。 The landmark feature was folder batch loading — and crucially, this was achieved on day one of development (March 28, 2026, V8). Moving from "select files one by one" to "point to a folder and instantly load 1,326 tracks" was the pivot that transformed CANTO from a prototype into a real tool.

Prompt / ネーミングと機能拡張の依頼 Prompt / Naming & Feature Request 
USER →このMP3プレーヤーに名前をつけたい。音楽・歌・詠唱に関連するラテン語や音楽用語で、プロフェッショナルかつ覚えやすい名前を提案してください。また、フォルダーを指定してその中の音楽ファイルをまとめてライブラリーに追加できるようにしてください。I want to name this MP3 player. Please suggest a professional yet memorable name related to music, song, or chanting — from Latin or music terminology. Also, please add the ability to select a folder and batch-load all music files inside into the library.
Response / Claudeのレスポンス抜粋 Response / Claude's Key Output
「CANTO — ラテン語で「私は歌う」。音楽プレーヤーとして完璧な名前です。シンプルで発音しやすく、音楽の本質を一言で表現しています。」 "CANTO — Latin for 'I sing.' A perfect name for a music player. Simple, pronounceable, and it captures the essence of music in a single word."

フォルダー機能については：
「📁 フォルダー追加機能を実装しました。input[webkitdirectory]を使用してフォルダーごと読み込みが可能になりました。これにより、大量の音楽ファイルを一括でライブラリに追加できます。」 On the folder feature:
"📁 Folder batch-add implemented using input[webkitdirectory]. Entire folders can now be loaded at once, allowing bulk import of large music collections."
